# BV产品需求池管理 — 功能说明文档

## 系统概述

BV产品需求池管理是一个轻量级的产品需求管理工具，以静态单页应用的形式部署在 GitHub Pages 上，无需服务器即可使用。

**访问地址**: `https://xiaoyao-2026.github.io/bv-requirements/`

---

## 数据存储机制

系统采用 **手动保存/云端同步机制**，实现多端一致性与数据安全：

| 层级 | 存储位置 | 说明 |
|------|----------|------|
| 内存层 | Vue 响应式数据 (`db.value`) | 所有新增、删除、编辑、状态流转操作都会即时更新在内存中，响应极为流畅 |
| 云端层 | GitHub Repository (JSON) | 点击右上角「💾 保存」按钮后，将当前数据一次性推送到云端后台库 |

**主要特性**：
1. **多端最后写入为准**：每次刷新或加载页面时，系统都会**直接从 GitHub 拉取云端最新数据**（自动加时间戳以绕过浏览器与 GitHub CDN 缓存），以保证多端协作时看到的是云端最新数据。
2. **手动保存按钮指示**：
   - 按钮显示为绿色 💾 **保存** 时，表示当前内存数据与云端已同步。
   - 按钮高亮显示为橙黄色 💾 **保存 ●** 并带有红点时，表示本地存在未保存的修改，提醒用户同步到云端。
3. **安全覆盖逻辑**：保存操作会在发送 PUT 请求前动态拉取最新的 commit SHA，确保多次保存之间不会因为 SHA 冲突导致失败。

## 功能模块

### 1. 产品线管理（左侧面板）

| 功能 | 操作方式 |
|------|----------|
| **查看全部产品** | 点击「☰ 全部产品」查看所有需求 |
| **按产品筛选** | 点击具体产品线名称，仅显示该产品的需求 |
| **新增产品线** | 点击标题栏「＋ 新增」按钮，输入名称后创建 |
| **重命名产品线** | 鼠标悬停在产品线上，点击铅笔图标（✎），输入新名称保存 |
| **删除产品线** | 鼠标悬停在产品线上，点击垃圾桶图标，确认后删除（含其下所有需求） |

> ⚠️ 删除产品线会级联删除该产品线下的所有需求，操作前会弹出二次确认。

### 2. 需求管理（主内容区）

#### 2.1 需求卡片

每张卡片展示一条需求的核心信息：
- **标题**：功能名称
- **描述**：功能说明（最多显示4行）
- **优先级**：P0（紧急）、P1（高）、P2（中）、P3（低），以颜色标签区分
- **状态**：可在卡片上直接通过下拉框切换，修改即时保存
- **版本号**：所属迭代版本
- **计划日期**：计划完成时间
- **负责人**：产品负责人 / 研发负责人

#### 2.2 需求操作

| 功能 | 操作方式 |
|------|----------|
| **新增需求** | 点击右上角「＋ 新增需求」按钮 |
| **编辑需求** | 点击需求卡片，弹出编辑表单 |
| **修改状态** | 直接在卡片上的状态下拉框中选择，即时保存 |
| **删除需求** | 点击卡片底部的垃圾桶图标，确认后删除 |
| **批量选择** | 勾选卡片右上角的复选框 |
| **批量删除** | 选中后点击顶部「批量删除」按钮 |

#### 2.3 筛选与搜索

| 筛选器 | 说明 |
|--------|------|
| 版本筛选 | 按所属迭代版本过滤 |
| 优先级筛选 | 按 P0-P3 过滤 |
| 状态筛选 | 按需求状态过滤 |
| 关键词搜索 | 模糊搜索名称和说明 |
| 隐藏已完成 | 勾选后隐藏状态为「完成」的需求（默认开启） |

### 3. 禅道集成

系统支持将需求推送至禅道项目管理系统：

| 功能 | 说明 |
|------|------|
| **禅道配置** | 点击右上角「⚙ 禅道配置」，填写禅道地址、用户名、密码和默认产品ID |
| **单条推送** | 点击卡片底部的上箭头图标（↑），选择禅道产品后推送 |
| **批量推送** | 勾选多条需求后，点击「批量推送禅道」按钮 |

> 注意：禅道推送功能需要在本地运行代理服务 `node zentao-proxy.js`，系统会自动检测代理状态。

## 图标说明

系统使用统一的线性 SVG 图标：

| 图标 | 含义 | 使用位置 |
|------|------|----------|
| ✎ 铅笔 | 编辑/重命名 | 产品线、需求卡片 |
| 🗑 垃圾桶 | 删除 | 产品线、需求卡片 |
| ↑ 上箭头 | 推送到禅道 | 需求卡片 |
| ✕ 叉号 | 关闭弹窗 | 所有对话框 |

所有图标在鼠标悬停时显示对应的语义颜色（蓝色=编辑、红色=删除、绿色=推送）。

## 状态流转

需求状态支持以下值，可在卡片上直接切换：

```
未开始 → 需求整理 → 设计中 → 开发中 → 测试中 → 完成
```

每个状态以不同的颜色标签展示：
- **未开始**：灰色
- **需求整理 / 设计中**：紫色
- **开发中**：黄色
- **测试中**：蓝色
- **完成**：绿色
